‚≠ê ES Modules vs CommonJS
ES Modules (Recommended Modern Syntax)

Use import and export

Example:

import fs from "fs";
export function test() {}

CommonJS (Older Syntax)

Uses require() and module.exports

Example:

const fs = require("fs");
module.exports = { test };

1Ô∏è‚É£ File System Module (fs)
Purpose

Used to interact with files and folders in your local machine.

Use Cases

Read / write / update / delete files

Manage folders

File uploads & logs

Config file operations

Work with CSV, JSON data

Store small data without a database

Two Types

Asynchronous (Non-blocking) ‚Üí Fast, recommended

Synchronous (Blocking) ‚Üí Used when execution must stop

readlink()	Read symbolic link

rm()		Remove a file
unlink()	Delete a file

Character Encoding

UTF-8 ‚Üí Unified Transformation Format

Standard encoding format used in most file operations.

‚≠ê Very Short Summary for Interview (You can speak this)

Node.js provides core modules like fs, os, path, process and env.
The fs module allows reading, writing, updating, deleting files and folders.
OS gives system-level information, Path helps handle file paths in a cross-platform way, and Process helps access environment variables and control node execution.
I use ES Modules (import/export) because Node now supports them natively.
