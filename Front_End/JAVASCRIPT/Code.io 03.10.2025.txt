ðŸ§© Topic: Logical Operators, Nullish Coalescing, and Iteration Statements in JavaScript
ðŸ”¹ 1. Logical Operators

Logical operators are used to combine or invert boolean values.

AND (&&)

Returns true only if all operands are truthy.

Stops evaluating as soon as it finds false (short-circuit).

console.log(true && true);   // true

OR (||)

Returns the first truthy value.

If all are falsy, returns the last value.

Does not always return boolean â€” returns actual value.

console.log(null || 0 || "Hello"); // "Hello"
console.log(null || 0 || undefined); // undefined

NOT (!)

Inverts a boolean value.

console.log(!true);  // false
console.log(!false); // true

ðŸ”¹ 2. Nullish Coalescing Operator (??)

Assigns a value only if the variable is null or undefined.

Does not overwrite other values (like 0 or empty string).

console.log(current_age); // 19

current_age = age ?? 19;
console.log(current_age); // 18


âœ… Use case: Default values for uninitialized variables.

General Rule: Initialization; Condition; Update

Checks condition before executing the loop.

b) do...while Loop (Exit-controlled)

Executes at least once, then checks condition.

c) for Loop

    console.log(num);

ðŸ§© Topic: Jump Statements and Switch Case in JavaScript
ðŸ”¹ 1. Jump Statements

Jump statements are used to control the flow of loops.

a) continue

Skips the current iteration and moves to the next iteration of the loop.

for (let i = 1; i <= 5; i++) {
    if (i === 3) {
        continue; // skip 3
    }
    console.log(i);
}
// Output: 1 2 4 5

b) break

Stops the entire loop immediately.

let n = 4;
let ans = 0;

for (let i = 1; i <= 4; i++) {
    if (i === 2) {
        break; // loop stops here
    }
    ans += i; // ans = ans + 1
}
console.log(ans); // 1


ðŸ“˜ Explanation:

Loop breaks at i === 2.

Only the first value (1) is added before breaking.

ðŸ”¹ 2. Switch Case

switch is used for multiple conditions on the same variable.

Similar to multiple else if statements.

Each case is compared using strict equality (===).

Syntax:

switch(variable) {
    case value1:
        // code
        break;
    case value2:
        // code
        break;
    default:
        // code if no match
        break;
}

Example Task: Print Number in Words
let n = 5;

switch(n) {
    case 1:
        console.log('one');
        break;
    case 2:
        console.log('two');
        break;
    case 3:
        console.log('three');
        break;
    case 4:
        console.log('four');
        break;
    case 5:
        console.log('five');
        break;
    default:
        console.log('Number greater than 5');
        break;
}


ðŸ–¥ï¸ Output:

five

ðŸ”¹ Key Notes

break â†’ stops loop or switch case

continue â†’ skips current iteration in a loop

switch â†’ better readability for multiple conditions

Always use break in switch to avoid fall-through

default â†’ executes if no case matches

ðŸ”¹ Interview Questions

Q: What is the difference between break and continue?
A: break stops the loop completely; continue skips current iteration and moves to the next.

Q: What happens if you donâ€™t use break in a switch case?
A: Code will fall-through and execute subsequent cases until a break is encountered.

Q: Can switch cases handle ranges (like n > 5 && n < 10)?
A: âŒ No, switch only checks exact values. Use if-else for ranges.

Q: What is the purpose of the default case?
A: It runs when none of the cases match the variable value.

Q: Can switch be used with strings?
A: âœ… Yes, switch works with numbers, strings, and even booleans.

ðŸ”¹ 1. Value vs Reference
Value Types (Primitive)

Stored directly in memory

Examples: Number, String, Boolean, undefined, null, BigInt, Symbol

Assignment creates a copy of the value

let x = 10;
let y = x;
y = 20;
console.log(x, y); // 10 20

Reference Types (Objects)

Stored as reference in memory

Examples: Object, Array, Function

Assignment points to same memory location, so changes affect all references

let a = { name: "React", year: 2025 };
let b = a;

console.log(a, b); 
// {name: "React", year: 2025} {name: "React", year: 2025}

b.year = 75;
console.log(a, b); 
// {name: "React", year: 75} {name: "React", year: 75}


âœ… Explanation: b is a reference to a, so changing b.year changes a.year too.

JavaScript automatically frees memory that is no longer referenced

Helps prevent memory leaks

ðŸ”¹ 3. Constructor Functions

Used to create multiple similar objects easily

Naming convention: First letter capitalized

this keyword refers to the object being created

new keyword creates a new object from the constructor

Syntax
function FunctionName() {
    this.property = value;
    this.method = function() { ... }
}

let obj = new FunctionName();

Example 1: User Constructor
function User() {
    this.name = "Jagath";
}

let jagath = new User();
let siva = new User();

console.log(jagath.name); // Jagath
console.log(siva.name);   // Jagath

Example 2: Add Constructor
function Add() {
    this.a = 5;
    this.b = 10;
    this.add = function(c) {
        return this.a + this.b + c;
    }
}

let sum = new Add();
console.log(sum.add(11)); // 26


âœ… Explanation: this.a and this.b are properties of the object sum. add is a method using this.

Q: What is the difference between value and reference in JavaScript?
A: Value types are copied independently; reference types share memory.

Q: What is garbage collection in JavaScript?
A: Automatic memory management that frees memory no longer in use.

Q: What is a constructor function?
A: A function used to create multiple objects with the same structure.

Q: What does this refer to in a constructor function?
A: The object being created.

Q: How do you create a new object using a constructor?
A: Using the new keyword: let obj = new ConstructorName();

Q: What happens if you modify a property of a reference type?
A: All references to that object reflect the change.
