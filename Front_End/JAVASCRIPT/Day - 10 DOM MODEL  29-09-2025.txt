ğŸŒ Document Object Model (DOM)

ğŸ”¹ Definition

The DOM is the bridge between the frontend (HTML, CSS, Bootstrap) and JavaScript.

When a webpage loads, the browser creates a tree structure (object model) of all HTML elements.

Each HTML element is represented as a node in this tree.

JavaScript can access, modify, create, or remove these nodes dynamically using the DOM.

The DOM is not part of JavaScript. It is a Browser API that JavaScript uses.

ğŸ‘‰ Formula: HTML + Document (API) + JavaScript = Interactive Webpage

ğŸ”¹ Disadvantage

When you change even one small part of the DOM, the browser may re-render or refresh the whole page layout, which can affect performance.

ğŸ”¹ Example
console.log(document.body); // Prints the body content of the page

âœ… DOM Interview Questions & Answers
Q1. What is the DOM?

Answer:
The DOM (Document Object Model) is a programming interface for HTML and XML documents. It represents the document as a tree structure, where each node is an object representing part of the page (elements, attributes, text). JavaScript can use the DOM to dynamically change content, structure, and styling of a webpage.

Q2. Is DOM part of JavaScript?

Answer:
No, the DOM is not part of JavaScript. It is a Browser API provided by the browser. JavaScript can use this API to manipulate HTML and CSS.

Q3. What are DOM Selectors?

Answer:
DOM selectors are methods to select elements from the document. Examples:

getElementById("id")

getElementsByClassName("class")

querySelector("selector")

querySelectorAll("selector")

Q4. Difference between innerHTML, innerText, and textContent?

Answer:

innerHTML: Gets/sets HTML including tags.

innerText: Gets/sets only visible text (ignores hidden text & tags).

textContent: Gets/sets all text, including hidden ones, without parsing HTML.

Q5. How do you create and insert a new element using the DOM?

Answer:

const div = document.createElement("div");
div.innerText = "New Element";
document.body.appendChild(div);

Q6. What is the difference between == (inline event handlers) and addEventListener?

Answer:

Inline / old way (onclick): You can attach only one handler at a time. If you assign a new one, the old one is replaced.

addEventListener: You can attach multiple handlers to the same event, and it is the modern recommended method.

Q7. What are common DOM events?

Answer:

Mouse events â†’ click, mouseover, mouseout

Keyboard events â†’ keydown, keyup, keypress

Form events â†’ submit, change, focus

Q8. What is event bubbling and event capturing?

Answer:

Event Bubbling (default): Event starts from the target element and bubbles up to parent elements.

Event Capturing: Event starts from the root/parent and goes down to the target element.
ğŸ‘‰ You can control this by using addEventListener(event, handler, useCapture).

Q9. What is the difference between NodeList and HTMLCollection?

Answer:

NodeList: Returned by querySelectorAll(). Can contain element nodes, text nodes, etc. It is static (doesnâ€™t auto-update).

HTMLCollection: Returned by getElementsByClassName(), getElementsByTagName(). Contains only element nodes and is live (updates automatically when DOM changes).

Q10. What are the disadvantages of the DOM?

Answer:

Performance issues: Changing one node may cause the entire page to re-render.

Large DOM trees are expensive to update frequently.

DOM manipulations can slow down performance if not optimized.

Q11. How do you remove an element from the DOM?

Answer:

const element = document.getElementById("myDiv");
element.remove();  // Modern way
// OR
element.parentNode.removeChild(element); // Old way

Q12. How does the DOM differ from the Virtual DOM (used in React)?

Answer:

DOM: Direct representation of the webpage. Updates are costly since they re-render the page.

Virtual DOM: A lightweight copy of the DOM (in memory). Only the changed parts are updated, improving performance.
